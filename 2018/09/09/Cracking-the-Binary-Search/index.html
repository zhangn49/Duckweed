Cracking the Binary Search

Sep 9, 2018

In computer science, Binary search is one of the most famous algorithms. It is a search algorithm that finds the position of target value within a sorted array. It compares the middle value to the target value, and narrow down the search range until the value is found or range becomes zero.

It looks like very straightforward, but writing a bug free binary search is not simple. Accoding to the Programming Pearls,

given ample time, only about ten percent of professional programmers were able to get this small program right.

Acutally, the binary search is not restricted to searching an element in a sorted sequence. There are various forms of Binary Search:

  • Find the target value k
  • Find the first value larger than k
  • Find the first value larget than or equal to k
  • Find the first value less than k
  • Find the first value less than or equal to k
  • Find the frst value equal to k
  • Find the last value equal to k

Fortunately, behind these different questions, there is an principle that remains true, no matter which version of questions we are facing. The principle is using a pointer as boundary and move another to find the target.

Before I start to show how to apply this principle to questions, I need to clarify that I always use [start, end] to represent an array, which means num[start] and num[end] are inclusive. For the pointers, I prefer to use left and right to indicat their relative positions. For the result, it will be the index of the given array, or -1 if the target does not exist.

Find the first value larger than k

For this case, we scan from start and minimize the searching range. If the mid value is less than or equal to k, it is not our wanted value and we will drop it.

Find the first value larget than or equal to k

For this case, we scan from start and minimize the searching range. If the mid value is less than k, it is not our wanted value and we will drop it.

Find the first value less than k

For this case, we scan from end and minimize the searching range. If the mid value is larger than or equal to k, it is not our wanted value and we will drop it.

Find the first value less than or equal to k

For this case, we scan from end and minimize the searching range. If the mid value is larger than k, it is not our wanted value and we will drop it.

Find the frst value equal to k

For this case, we scan from start and minimize the searching range. If the mid value is less than k, it is not our wanted value and we will drop it. Check the left and array[left]!

Find the last value equal to k

For this case, we scan from end and minimize the searching range. If the mid value is larger than k, it is not our wanted value and we will drop it. Check the right and array[right]!
